# Antarctica

Q1: A Python desktop application for extracting and exporting GitHub repository data into Excel spreadsheets. The application provides a simple graphical user interface built with CustomTkinter and includes AI-powered functionality using Google's Gemini API.

Q2 Analysis of fund data. Calculation of alphas, betas, sharpe ratio etc. Analysis to determine if the fund performes better than the underlying assets

## Features

* Q1:
* Search for GitHub users
* Retrieve public repository name, url, README url
* Export repository data to Excel
* desktop GUI using CustomTkinter
* Gemini AI integration

* Q2:
* Regression model fitted to data
* Alpha and Beta calculations
* Sharpe ratio calculations of fund vs underlying assets
* Comparison of betas in different timeframes of the fund

## Prerequisites

Before running the application, ensure the following are installed:

* Python 3.11+ (recommended, 3.11.13 used)
* Git

Check your installation:

```bash
python --version
git --version
```

---

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/AlfieRootham/Antarctica.git
```

### 2. Navigate to the Project Directory

```bash
cd Antarctica
```

### 3. Create a Virtual Environment

Windows:

```bash
python -m venv venv
```

Mac/Linux:

```bash
python3 -m venv venv
```

### 4. Activate the Virtual Environment

Windows Command Prompt:

```bash
venv\Scripts\activate
```

Windows PowerShell:

```powershell
.\venv\Scripts\Activate.ps1
```

Mac/Linux:

```bash
source venv/bin/activate
```

You should now see `(venv)` at the beginning of your terminal prompt.

---

## Install Dependencies

Install all required packages:

```bash
pip install -r requirements.txt
```

---

## Environment Variables

The application requires a Google Gemini API key.

Use the file called example_env in the project root directory:
Re name it to .env

```env
GEMINI_API_KEY=YOUR_API_KEY_HERE
```

Replace `YOUR_API_KEY_HERE` with your Gemini API key.

---

## Running the Application

Run the application from the project root:

```bash
python Q1.py
```

If the main file is located elsewhere, adjust the command accordingly.

Examples:

```bash
python src/Q2.py
```

or

```bash
python Q1.py
```

---

## Updating Dependencies

If additional packages are installed during development, update the requirements file:

```bash
pip freeze > requirements.txt
```

## Notes

* The `.env` file is intentionally excluded from source control and should not be committed to GitHub.
* Excel files generated by the application are stored locally on the user's machine.
* Before you try and export to an excel file you must first create a blank one in your desired location
* The application uses publicly available GitHub repository information.
